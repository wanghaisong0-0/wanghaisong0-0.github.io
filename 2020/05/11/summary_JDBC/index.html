摘要：

JDBC让我们可以用Java语言操作数据库，开发者通过JDBC(接口、规范),只需要编写一种代码，通过每个厂家提供的jar包(驱动)，运行代码，执行厂商的实现类。连接池(Druid)对JDBC进行了封装，提高了连接的复用性，节约资源，减轻服务器压力。

一、JDBC基础

1.概述

Java 数据库连接（Java DataBase Connectivity）

作用：通过Java语言操作数据库

操作实现步骤：

  1. 准备数据库和表

  2. 创建java工程，导入MySQL驱动jar包

  3. 编写插入代码

DriverManager：驱动管理对象

Connection：数据库连接对象

Statement：执行sql的对象

ResultSet：结果集对象,封装查询结果

4.PreparedStatement

使用Statement会产生SQL注入问题

解决sql注入问题

我们就不能让用户输入的信息和SQL语句进行字符串拼接。需要使用PreparedSatement对象解决SQL注入。

PreparedSatement基础语法

优点

  1. 防止sql注入，提高安全性
  2. 参数与sql分离，提高代码可读性
  3. 减少编译次数，提高性能

二、连接池(Druid)

连接池其实就是一个容器(集合)，存放数据库连接的容器。

优点

  • 节约资源，减轻服务器压力
  • 提高连接复用性，用户访问高效

实现

Java为数据库连接池提供了公共的接口: DataSource ，各个连接池厂商去实现这套接口，提供jar包。
